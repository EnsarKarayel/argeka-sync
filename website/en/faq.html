FAQ

Short answers about installation, data safety and usage.

Is ARGEKA Sync a cloud service?

No. The website introduces the product and provides the setup download. The app runs on the user's own Windows computer or server.

Where do database passwords go?

Connection details stay in the local installation environment. ARGEKA Sync does not send them to a central SaaS system.

Which databases are active?

PostgreSQL/Internal, Microsoft SQL Server and MySQL/MariaDB are active engines. Oracle, ODBC, CSV/Excel, REST API and SAP HANA are planned connector types in the architecture.

Is an account required after installation?

For normal local installation, no web account is required. The panel starts a local session through localhost automatically.

Does the download start automatically?

No. For Google Ads and user safety, downloads do not start when the page opens. The setup file downloads only after the user clicks the download button.

What should I do before connecting production data?

Test with a test table or database first, take a target backup and validate column mapping with a small data set.
